# KOL 系统重构计划

> 目标：统一搜索→采集→粗估→细估四步流水线，理清表职责和关联逻辑。

## 一、统一流水线

```
Step 1: 搜索           Step 2: 信号采集       Step 3: 落库+粗估         Step 4: 细估
(skill调参数)           (脚本自动)             (脚本自动)                (skill+人工)
     │                      │                       │                        │
     ▼                      ▼                       ▼                        ▼
yt_search.py           youtube_data.py       ┌──────────────┐         ┌──────────────┐
+时间筛选              → sig.json            │ 平台明细表    │         │ 评估记录表    │
+发布时间范围                                │ (数据+浅判断) │         │ (5维评分+报价) │
                                             └──────┬───────┘         └──────┬───────┘
                                                    │                        │
                                              Agent 粗估                     │
                                               ┌────┴────┐                  │
                                           [通过]      [淘汰]                │
                                               │    标记原因                 │
                                               ▼                            │
                                        ┌──────────────┐                    │
                                        │ 候选池        │                    │
                                        │ (待细估工作台) │── 提醒对接人 ────▶ │
                                        └──────┬───────┘                    │
                                               │                            │
                                               └─── 关联主体 ──────────────▶ │
                                                                             ▼
                                                                      ┌──────────────┐
                                                                      │ KOL总表       │
                                                                      │ (Hub 聚合)    │
                                                                      └──────────────┘
```

### 核心变化
- 粗估 = 全自动硬指标筛选，不需要 Agent
- Agent 只在细估阶段介入（5维深度评分 + 报价建议）
- 搜索和手动输入走同一条管道
- 候选池是对接人的工作台，不再是搜索的暂存区

## 二、表职责划分

| 表 | 定位 | 写入时机 |
|---|---|---|
| **平台明细表** (YouTube/TikTok/IG/TW) | 数据仓库 — 所有被采集过的 KOL 原始数据 + 浅判断 | Step 3 信号落库 |
| **候选池** | 工作台 — 粗估通过、等待细估的 KOL | Step 3 粗估通过后 |
| **评估记录** | 判断卡 — 细估的完整 5 维评分 | Step 4 细估完成后 |
| **KOL 总表** | Hub — 一人一行，聚合全局状态 | Step 3 建/挂主体；Step 4 聚合回填 |

## 三、信号采集覆盖（Step 2 产出 sig.json）

### YouTube (youtube_data.py)
- 频道：名称、channel_id、URL、头像、订阅数、视频总数、创作者国家、分类、邮箱、最近更新日期
- 指标：加权 ER、赞率、评论率、均播放、中位播放、更新间隔天
- 评论：泛灌水占比、模板化重复占比、0赞占比、评论语言分布、购买意图信号
- 近期热门视频 (标题/播放/日期/URL)

### TikTok (tiktok.py)
- 频道：名称、sec_uid、unique_id、粉丝数、总赞、视频数、地区、签名
- 指标：加权 ER、赞率、评论率、转发率、均播放、中位播放、播放粉丝比
- 评论：语言分布、购买意图数、重复评论数

### Instagram (instagram.py)
- 频道：名称、user_id、粉丝数、关注数、帖子数、是否商业号、分类、简介、外链
- 指标：互动率、均赞、均评

### Twitter (twitter.py)
- 频道：名称、rest_id、粉丝数、关注数、蓝V、认证类型、位置、简介
- 指标：加权 ER、点赞率、转推率、回复率、收藏率、均 views

## 四、粗估硬标准（Step 3 自动判定）

| 维度 | YouTube | TikTok | Instagram | Twitter | 判定 |
|---|---|---|---|---|---|
| ER 下限 | ≥ 2% | ≥ 3% | ≥ 1% | ≥ 0.3% | 低于 → 淘汰 |
| 评论质量 | 非"严重灌水" | 同左 | 同左 | 同左 | 灌水 → 一票否决 |
| 最近更新 | ≤ 90天 | ≤ 90天 | ≤ 90天 | ≤ 90天 | 超期 → 淘汰 |
| 粉丝下限 | ≥ 1K | ≥ 1K | ≥ 1K | ≥ 500 | 低于 → 淘汰 |
| 受众辐射 | 目标区(欧美/日本) | 同左 | 同左 | 同左 | 非目标区 → 标旗(不一票否决) |

粗估得分 = 各维度加权打分(0-100)，≥ 60 进候选池，< 60 淘汰并记原因。

### 关联方式

| 从 → 到 | 字段 | 类型 | 说明 |
|---|---|---|---|
| 平台明细 → KOL总表 | `所属KOL` | link(双向) | 每个平台账号挂到一个主体 |
| 候选池 → KOL总表 | `关联主体` | link(单向) | 粗估通过后建/挂主体 |
| 评估记录 → KOL总表 | `关联主体` | link(双向) | 细估判断卡挂到主体 |
| 候选池 → 平台明细 | `来源表` + `来源记录ID` | 文本 | 逻辑关联，追溯到原始数据行 |

### 每步数据流

```
Step 1: 搜索
────────────────────────────────────────────────────
输入: 用户指定关键词 + 时间范围
  │
  ▼
yt_search.py ──READ──▶ YouTube KOL明细(去重)
  │
  ▼
/tmp/yt_search_result.json (新频道列表)
  ·不写任何飞书表


Step 2: 信号采集
────────────────────────────────────────────────────
输入: @handle 或 channel URL
  │
  ▼
youtube_data.py / tiktok.py / instagram.py / twitter.py
  │
  ▼
/tmp/sig.json (频道指标 + 评论分析 + 商单信号)
  ·不写任何飞书表


Step 3: Agent 粗估 + 候选池写入 (write_candidate.py)
────────────────────────────────────────────────────
输入: /tmp/sig.json + --business + --source

  sig.json
    │
    ├──▶ 自动推导浅判断
    │      评论质量标记 ← 泛灌水占比 + 模板化重复占比
    │      受众辐射推断 ← 评论语言分布
    │
    ├──WRITE──▶ 平台明细表 (upsert by platform ID)
    │            · 客观数据: 粉丝/ER/均播放/国家/...
    │            · 浅判断: 评论质量标记/受众辐射
    │            · 粗估状态: 通过 or 淘汰
    │            · 粗估淘汰原因: (如有)
    │
    ├──▶ 硬标准打分 (0-100)
    │      ER ≥ 阈值? + 粉丝 ≥ 下限? + 活跃? + 不灌水?
    │
    ├──[通过 ≥60]──WRITE──▶ 候选池
    │                        · 复制关键字段(独立存储)
    │                        · 候选状态 = 待细估
    │                        · 来源表 + 来源记录ID(追溯)
    │               WRITE──▶ KOL总表 (建/挂主体)
    │
    └──[淘汰 <60]──▶ 平台明细表已标记(粗估状态=淘汰)
                      ·不写候选池


Step 4: 细估 (write_kol.py + sync_hub.py)
────────────────────────────────────────────────────
输入: /tmp/sig.json + /tmp/judge.json (Agent产出)

  sig.json + judge.json
    │
    ├──WRITE──▶ 平台明细表 (补充Agent判断字段)
    │            · 内容题材/提及工具/题材_LLM/调性_LLM
    │
    ├──WRITE──▶ 评估记录表 (新建判断卡)
    │            · 5维评分(1-10)
    │            · 综合判断/判断依据
    │            · 报价区间/计价口径
    │            · 关联主体 → KOL总表
    │
    └──sync_hub.py──READ──▶ 评估记录 (聚合)
                    WRITE──▶ KOL总表 (回填综合判断/情况说明/时间戳)
```
