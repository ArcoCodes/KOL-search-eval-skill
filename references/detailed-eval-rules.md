# KOL 细估规则说明

> 总表 5 个文本字段，Agent 细估时按以下指导分析，每个维度写一句话摘要。
> judge.json 中填字符串，write_kol.py 落库到 KOL 总表。

---

## 一、频道评估

> 回答"这个人是谁、为什么有影响力、频道是什么性质"。

### 需要判断的要素

| 要素 | 怎么看 | 数据来源 |
|------|--------|---------|
| 出镜形式 | 真人出镜 / AI 数字人 / 无脸(画外音+屏录) | 视频缩略图、视频内容 |
| 创作者背景 | 是真正的从业者还是内容搬运者？有没有可验证的专业经验？ | 简介、外链、视频深度 |
| 领域定位 | 在什么圈子有话语权？定位是否清晰稳定还是频繁换赛道？ | 频道分类、标签、历史内容主题 |
| 个人品牌 | 有没有自己的产品/课程/社区/newsletter？ | 简介外链、视频描述 |
| 频道真实性 | 是否有矩阵号/买号/养号的嫌疑？ | 频道创建时间 vs 内容量、风格是否雷同其他频道 |

### 分析指导

- **关注频道的"人设可信度"**：真人出镜+垂直深耕+有专业背景的频道，商单可信度远高于无脸聚合频道
- **AI 数字人频道要单独标注**：用 HeyGen/D-ID/Synthesia 等工具生成虚拟形象出镜的频道越来越多，需要和真人出镜区分——它们的"人设"是虚构的，受众信任度不同
- **识别内容农场信号**：频道名/封面/风格高度雷同其他频道（MCN 矩阵号）；频道很老但内容很少且最近突然活跃（买号）；无任何联系方式（一次性频道）
- **有自有品牌是双刃剑**：说明创作者有商业能力，但也意味着商单可能不是其优先级，报价可能更高

### 写法示例

- "英国独立创作者，AI 视频制作领域，真人出镜教程型，有自己的 coaching 业务"
- "AI 数字人频道，用 HeyGen 生成形象，内容为 AI 工具盘点，无真人背景"
- "无脸聚合频道，AI 新闻/工具盘点，风格雷同多个同类频道，矩阵号可能性大"

## 二、受众结构

> 回答"谁在看这个人的内容，是不是我们要触达的人"。

### 需要判断的要素

| 要素 | 怎么看 | 数据来源 |
|------|--------|---------|
| 受众地域 | 评论语言分布推断主要地域 | `评论语言分布(估算)`、创作者自报国家 |
| 受众类型 | 观众是开发者？创作者？学生？泛科技爱好者？ | 评论内容（问什么问题）、频道定位 |
| 业务线匹配 | 这群受众是否是 Bloome/Renoise 的潜在用户 | 结合 business-standards.md 目标画像 |
| 受众质量 | 评论是真讨论还是纯灌水？有没有购买意图？ | 评论采样内容、`purchase_intent_count` |

### 分析指导

- **地域是硬关切**：目标区 = 欧美 + 日本。受众落在非目标区（东南亚/南亚/拉美等）→ 需在摘要中明确标注
- **从评论内容反推受众画像**：
  - 问工具链/API/部署/价格 → 开发者/技术用户 → Bloome 高匹配
  - 问模板/效果/生成质量/批量场景 → 创作者/营销人 → Renoise 高匹配
  - "太厉害了""求更新"等泛泛夸赞 → 泛科技爱好者 → 曝光可以但转化弱
  - "怎么涨粉""怎么变现" → 社媒运营者 → 和 AI 产品关联弱
- **评论语言只是弱推断**，真实受众地域需 KOL 后台数据(media kit)验证，摘要中应注明推断的置信度
- **购买意图比例是强信号**：purchase_intent >5% 说明受众有真实需求；接近 0% 说明受众看热闹不买单

### 写法示例

- "欧美为主(100%英文)，受众是 AI 开发者/自动化玩家，评论常问 API 和部署，Bloome 高匹配"
- "分散(40%英文/25%西语/15%葡语)，受众偏泛科技爱好者，购买意图 <1%，转化预期弱"

## 三、流量稳定性

> 回答"合作那条视频大概能拿多少播放，排期是否可控"。

### 需要判断的要素

| 要素 | 怎么看 | 关键指标 |
|------|--------|---------|
| 更新频率 | 是否规律更新，排期是否可控 | `upload_interval_days_avg` |
| 播放稳定性 | 每条播放是否稳定，还是靠爆款拉高均值 | 均播/中位比：<2x 稳定，2-5x 有爆款，>5x 严重依赖爆款 |
| 播放/粉丝比 | 每条视频触达多少比例的粉丝 | >0.3 好，0.1-0.3 一般，<0.1 差 |
| 趋势方向 | 频道在上升、稳定还是衰退 | 近期视频播放 vs 历史均值 |

### 分析指导

- **均播 vs 中位是核心**：均播被爆款拉高是常见的，中位播放才代表"合作那条大概能拿多少"。如果均播 100 万但中位只有 5 万，合作那条很可能只有 5 万
- **识别四种典型模式并在摘要中点明**：
  - 稳定输出型（周更/双周更，播放波动小）→ 低风险，排期可控
  - 爆款驱动型（低产但偶尔出百万）→ 高风险，合作那条可能不爆
  - 衰退型（曾经高播放，近期下滑）→ 高风险，报价可能基于历史虚高
  - 新星型（视频不多但每条都在涨）→ 中风险，潜力大但样本少
- **更新间隔 >60 天的频道要重点标注排期风险**：无法保证在 campaign 周期内出视频
- **长尾 vs 前载**：教程/how-to 类内容有长尾搜索价值，新闻/热点类内容播放集中在前 48 小时

### 写法示例

- "周更稳定，中位播放 12K，均播 15K(均/中比 1.25x)，排期可控，播放可预估"
- "极低产(134天/条)，均播 96 万但仅 14 条视频，合作排期完全不可控，不适合有时间节点的 campaign"

## 四、互动真实性

> 回答"这些数据是真的吗，粉丝是真人吗，互动是自然的吗"。

### 需要判断的要素

| 要素 | 怎么看 | 关键指标 |
|------|--------|---------|
| ER 水位 | 互动率是否在平台正常范围 | YouTube 2-5%，TikTok 3-6%，IG 1-3%，Twitter 0.3-1% |
| 评论真实度 | 灌水/模板重复/0 赞评论的占比 | 泛灌水占比、模板重复占比、0 赞评论占比 |
| 评论者独立性 | 是不是同一批人在刷 | 独立评论者 / 总评论数，<80% 可疑 |
| 频道主互动 | 创作者是否回复评论 | `owner_reply_rate` |
| 购买意图 | 评论中有无真实需求 | `purchase_intent_count` |

### 分析指导

- **不同平台 ER 不能用同一把尺**：YouTube 5% 是优秀，但 Twitter 0.5% 也是优秀。用错基准会误判
- **重点排查以下造假模式**：
  - **买粉**：粉丝数 vs 中位播放严重倒挂（50 万粉中位播放 <1000）
  - **刷赞评**：ER 异常高（YouTube >15%）、赞评比失衡（自然比通常 20:1 到 50:1）、独立评论者 <80%
  - **买播放量**：播放高但赞评极少（赞评/播放 <0.5%）
  - **灌水评论**：模板重复 ≥15% 或泛灌水 ≥40%
- **ER 异常高也是红旗**：不要以为 ER 越高越好，远超平台上限可能是刷量
- **频道主回复是加分项**：owner_reply_rate >30% 说明创作者真的在和受众互动，社区感强
- **如果 like_count 数据缺失（yt-dlp 限制），要在摘要中注明 ER 不完整**

### 写法示例

- "ER 3.5% 健康(YouTube 范围内)，独立评论者 99%，模板重复 1%，频道主回复率 31%，互动真实可信"
- "ER 0.8% 偏低，泛灌水 35%，0 赞评论 52%，购买意图 0%——互动质量一般，受众参与度低"
- "粉丝 50 万但中位播放仅 800，疑似买粉；评论独立作者仅 65%，互动可信度低"

## 五、内容匹配度

> 回答"这个人的内容和我们的产品能不能自然结合"。

### 需要判断的要素

| 要素 | 怎么看 | 数据来源 |
|------|--------|---------|
| 题材相关性 | 内容主题和业务线关键词的重叠度 | 视频标题/标签/分类 vs business-standards.md |
| 内容形式 | 什么类型的内容？植入价值如何？ | 视频内容分析 |
| 作品质量 | 有无真实作品/demo，还是纯口播？ | 视频内容 |
| 竞品关系 | 和竞品的绑定程度 | `sponsor_intent`、affiliate 链接、`提及工具` |
| 植入可行性 | 产品能否自然融入其常规内容 | 基于内容形式和题材综合判断 |

### 分析指导

- **内容形式决定植入价值**（从高到低）：
  1. 真实 workflow 演示 → 观众跟做，转化路径最短
  2. 深度测评/对比 → 有说服力，但可能提到竞品
  3. 挑战/实验型 → 有趣但需包装成自然场景
  4. 教程/How-to → 产品必须是教程核心工具才有价值
  5. 新闻/盘点/十大工具 → 曝光广但停留浅，转化差
  6. 纯娱乐/梗 → 播放可能高但和产品零关联

- **竞品绑定是关键风险**：
  - 从未提及竞品或仅偶尔提到 → 白纸一张，最佳
  - 测评过竞品但无持续合作 → 可接受，说明关注赛道
  - 和竞品做过 1-2 次赞助 → 需确认排他协议
  - 是竞品 ambassador/affiliate → 高风险，大概率有排他

- **业务线匹配速查**：
  - Bloome：最佳 = Agent/workflow demo、AI coding 教程；不匹配 = AI 新闻、模型八卦
  - Renoise：最佳 = 成片对比、视频制作流程、批量广告；不匹配 = 纯工具罗列无作品

- **有真实作品是强加分**：做出过成片/跑通过代码/搭建过系统的创作者，比纯口播的可信度高几个量级

### 写法示例

- "AI 视频制作教程型，展示完整 workflow(Runway+Premiere)，有真实成片，Renoise 可自然植入"
- "AI 新闻盘点频道，每期罗列 10 个工具一句带过，无深度演示，植入价值低"
- "内容高度对口(AI coding workflow)，但已是 Cursor ambassador，竞品排他风险高"

## 通用原则

1. **每个维度写一句话摘要**：写清楚结论 + 最关键的依据，不要罗列数据。对接人扫一眼就能判断
2. **不确定的要标注**：如"评论语言推断，需 media kit 验证"、"like 数据缺失，ER 可能不准"
3. **业务匹配 ≠ 该合作**：题材对口只是入场券，身份/受众/互动任何一项有硬伤都应在摘要中说清
4. **平台差异必须考虑**：YouTube 的 5% ER 是优秀，Twitter 的 0.5% ER 也是优秀，不能一把尺量
