# KOL 粗估规则（Step 3）

> 执行者：Agent。粗估不再由 `scripts/screen.py` 自动完成。
> 输入：Step 2 采集到的 signals JSON（stdout 或临时文件）。
> 输出：Agent 产出粗估判断 JSON，再通过 `scripts/write_candidate.py` 写入候选池（通过=待细估，淘汰=已淘汰(浅筛)）。

---

## 1. 

粗估前，Agent 先从频道的评论数据推导两个浅判断字段：

### 评论质量标记

**YouTube**（基于 `data_scrawl/youtube_data.py` 的 `comments.signals` 输出）：

| 条件 | 标记 |
|------|------|
| 模板化重复评论占比 ≥ 15% **或** 泛泛灌水占比 ≥ 40% | 严重灌水/疑似养号 |
| 泛泛灌水占比 ≥ 20% | 一般 |
| 以上都不满足 | 真实讨论 |

**TikTok / Instagram / Twitter**（基于各平台评论采样）：

| 条件 | 标记 |
|------|------|
| 重复评论数 / 总评论数 ≥ 15% | 严重灌水/疑似养号 |
| 有评论数据且不满足上条 | 真实讨论 |
| 无评论数据 | 未知/待核实 |

### 受众结构（推断）

从评论语言分布推导（YouTube 取 `评论语言分布(估算)` 字段，其他平台取 `language_dist`）：

| 条件 | 推断结果 |
|------|---------|
| 英语/拉丁/法语/德语系 ≥ 50% | 欧美为主 |
| 日语/假名 ≥ 40% | 日本为主 |
| 葡萄牙语 ≥ 40% | 巴西/葡语拉美为主 |
| 英语 + 日语合计 < 25% | 非欧美为主 |
| 以上都不满足 | 分散无主导 |
| 无语言分布数据 | 未知/待核实 |

## 2. 硬标准打分（0-100 分）

### 平台阈值

| 平台 | ER 及格线 (`er_min`) | 粉丝最低线 (`subs_min`) |
|------|:-------------------:|:----------------------:|
| YouTube | 2% | 1,000 |
| TikTok | 3% | 1,000 |
| Instagram | 1% | 1,000 |
| Twitter/X | 0.3% | 500 |

不活跃天数上限：**90 天**（所有平台相同）

### ER 维度（0-30 分）

| 条件 | 得分 | 标旗 |
|------|:----:|------|
| ER ≥ `er_min` × 2 | 30 | — |
| ER ≥ `er_min` | 20 | — |
| ER ≥ `er_min` × 0.5 | 10 | "ER 偏低" |
| ER < `er_min` × 0.5 | 0 | "ER 过低" |
| ER 数据缺失 | 10 | "ER 数据缺失(待核实)" |

> 举例：YouTube `er_min`=2%，则 ≥4% 得 30 分，≥2% 得 20 分，≥1% 得 10 分，<1% 得 0 分。

### 粉丝数维度（0-20 分）

| 条件 | 得分 | 标旗 |
|------|:----:|------|
| ≥ 500K | 20 | — |
| ≥ 50K | 15 | — |
| ≥ 10K | 10 | — |
| ≥ 平台最低线（`subs_min`） | 5 | — |
| < 平台最低线 | 0 | "粉丝数过少" |
| 未知 | 0 | "粉丝数未知" |

### 活跃度维度（0-20 分）

基于最近一次更新距今天数：

| 条件 | 得分 | 标旗 |
|------|:----:|------|
| ≤ 30 天 | 20 | — |
| ≤ 60 天 | 15 | — |
| ≤ 90 天 | 10 | — |
| > 90 天 | 0 | "不活跃(最近更新X天前)" |
| 日期未知 | 10 | —（兜底） |

### 评论质量维度（0-30 分）

| 评论质量标记 | 得分 | 说明 |
|------------|:----:|------|
| 真实讨论 | 30 | — |
| 一般 | 15 | — |
| 严重灌水/疑似养号 | 0 | **触发一票否决** |

---

## 3. 通过规则

- **通过线**：总分 ≥ **60 分**（满分 100）
- **一票否决**：评论质量 = "严重灌水/疑似养号" → 无论总分多高直接淘汰
- **受众辐射标旗**：受众 = "非欧美为主" 或 "分散无主导" 时**不一票否决**，但标旗 "非目标区，需关注"

---

## 4. 粗估后的写入逻辑

| 粗估结果 | 平台明细表 | 候选池 | KOL 总表 |
|---------|-----------|--------|---------|
| **通过**（≥60 且未被否决） | **不写** | 写入，候选状态="待细估" | **不写** |
| **淘汰**（<60 或一票否决） | **不写** | 写入，候选状态="已淘汰(浅筛)" | **不写** |

### 去重键

| 平台 | 去重字段 |
|------|---------|
| YouTube | `主页URL` |
| TikTok | `sec_uid` |
| Instagram | `用户ID` |
| Twitter | `rest_id` |

已存在的候选池记录会更新而非重复创建。

### Agent 粗估判断 JSON

Agent 必须先把粗估结论写入 `/tmp/rough_judgment.json`：

```json
{
  "候选状态": "待细估",
  "粗估得分": 60,
  "评论质量标记": "一般",
  "受众辐射(推断)": "欧美为主",
  "粗估依据": "硬分60/100(ER20+粉丝15+活跃10+评论15); 评论泛灌水偏高但仍有真实问题讨论; 内容与 Renoise 的 AI 视频创作场景相关",
  "淘汰原因": "",
  "业务线": "Renoise"
}
```

字段要求：
- `候选状态` 只允许 `待细估` 或 `已淘汰(浅筛)`
- `粗估依据` 必须写具体内容，至少包含分项分数和关键通过/淘汰原因
- 淘汰时 `淘汰原因` 必须写清楚，不要只写“未达标”

写入候选池：

```bash
python3 scripts/write_candidate.py --signals /tmp/sig.json --judgment /tmp/rough_judgment.json --source "discovery:<keywords>" --keyword "<keywords>"
```

## 6. 当前边界

- Step 3 是 Agent 粗估，但仍只做浅筛，不做 5 维细估和报价
- Step 3 只决定是否进入候选池，不做 5 维评分和报价
- Step 3 推荐把 stdout JSON 保存到 `/tmp/sig.json`，便于 Agent 审阅并传给写入脚本
- 平台明细表、KOL 总表、评估记录都在 Step 4 细估阶段再写
