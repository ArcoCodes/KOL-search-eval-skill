# KOL 评估方法论（可复用经验沉淀）

> 本文沉淀从实战中校准出的 KOL 价值评估方法、有效/无效信号、以及踩过的坑。
> 目标：让后续评估（无论人还是 Agent）都能直接复用，不重复犯错。
> 配套脚本：`scripts/data_scrawl/youtube_data.py`（YouTube 统一信号采集入口）。

---

## 0. 第一原则：没有定义过的标准，不下"好/坏"结论

- Agent **不许**说"ER 很健康""粉丝质量好"这类结论，**除非**能引用一个已定义的标准（阈值 / 对照基准）。
- 没有标准时，正确表述是："指标 = X，目前无可对照标准，需先定义"。
- 所有判断必须**带分解指标 + 依据**，让人能质问、能推翻（人在环纠偏）。
- 行业经验阈值（如"评论/点赞 < 0.1 正常"）只是**待验证假设**，不是我们的标准；等自有库数据 + 转化回流后再校准。

## 1. Engagement Rate：

### 1.1 按播放加权，不要简单平均
- **简单平均**（每条 ER 取平均）会被低播放视频顶高 → 高估。
- **加权 ER** = 总(赞+评) / 总播放 ← **用这个**。

### 1.2 拆开看，别揉成一个数
- **点赞率**（赞/播放）= 被动认可，水分大、易刷。
- **评论率**（评论/播放）= 主动投入，更接近真实意愿 / 社群感。
- 对**转化**而言，评论率 + 评论质量大概率比点赞率更有预测力。

### 1.3 ER 当前的真正用途 = 反作弊 / 质量过滤，不是人气衡量
- 我们的北极星是拉新/转化；ER 只是"受众活跃度"的代理，**在拿到真实转化数据回测前只是假设**。
- ER 现在最实用的产出是帮初筛**筛掉刷量假号**。

### 1.4 反作弊信号（按可靠性排序）
| 信号 | 含义 | 可靠性 |
|------|------|--------|
| **模板化/重复评论** | 同一句话被 ≥2 个不同账号发出 = 复制粘贴 / AI 灌评论。客观、几乎无法被无辜解释 | ★★★ 最强 |
| **同批账号跨视频刷** | 固定账号在每条视频下重复出现 = 养号群 | ★★★ |
| **播放方差异常** | 真频道播放有高有低(CV 高、有扑街)；**全是爆款且播放量都差不多**(CV 低、几乎无低播放) = 买量嫌疑。**注水常只在赞助/投放窗口出现**，须按合作窗口单独查；且基线随品类变，宜相对同类 | ★★ |
| **泛泛灌水占比** | "Nice video""Very informative"等无具体内容的夸赞占比高 | ★★ |
| **真实观众行为缺失（语言学）** | 真观众会**提问/发表情/带时间戳**；养号多打磨过的**陈述句**。**⚠️软信号·仅作"该人工细看"提示灯**：受品类(科技/职场天然多陈述句)、受众成熟度、top 采样偏差影响大，**不可单独定性**；须相对同类对照 + 人工确认 + 真实结果验证 | ★ 提示灯，非判决 |
| **作者跨视频重复** | 同一批账号在每条视频下刷 = 互赞群 | ★★ |
| **频道主 0 回复** | 从不与评论互动，配合上面几条 = 养号常态 | ★★ |
| 评论/点赞比异常高 | 远超经验值（>0.1）需警惕，但要结合上面判断 | ★★（启发式） |
| 播放/订阅比偏低 | 粉丝不活跃或订阅注水的弱信号 | ★ |
| 点赞-播放相关性 | 接近 +1 健康；接近 0/负 异常。**注意**：赞率随播放下降是正常现象，不要误判 | ★（易误读） |

## 1.5 账号统计：能拿什么 / 拿不到什么（对照 Modash 学习校准）

`data_scrawl/youtube_data.py` 现产出（对标 Modash 数据层，yt-dlp 能力内）：
- 体量：订阅、**同量级区间**、长视频数、Shorts 数、更新频率
- 播放：近期均/中位播放、播放/订阅比、采样窗口**近期 Top**、**月度趋势**(采样近似)
- 互动：**加权 ER**（首选）+ 简单 ER + **分解 赞率/评论率** + 平均赞/评

**yt-dlp 拿不到（必须诚实标注 missing_fields，需 Modash 类平台或 KOL后台真实受众数据）：**
- **频道总观看 total_views**：root/about/playlist-items0 均返回 None → 只能求和全部视频或外部平台。
- **受众画像**：性别/年龄/国家/语言占比（Modash 的核心付费优势）。
- **订阅增长历史曲线**：yt-dlp 无历史；只能用采样视频近似播放趋势。
- **关注它的知名账号 / 相似频道 lookalikes**：Modash 独有。
- 全站历史热门 Top（当前仅采样窗口内）、创作者本人性别/年龄。

**对照 Modash 的两条校准**：
1. **加权 ER 经验证更准**：Erik 实测 yt-dlp 加权 2.76% ≈ Modash 2.96%（简单平均 4.2% 偏高）。**坚持用加权。**
2. **Modash 对 YouTube 不做真实性/credibility（字段为空）**——它给漂亮 demographics 但**不判养号**。我们的**评论内容反作弊是 Modash 没有的差异化**，二者互补：Modash 强在画像数据，我们强在真实性判断 + 业务匹配 + 报价。

## 2. 受众地域：最关键也最拿不到的维度
- 真实受众地区是频道主私有数据（YouTube Studio），**任何第三方 API 都只能"推断"**，唯一真值来源是 KOL 提供的 KOL后台真实受众数据（放在邮件沟通环节问）。
- 可用的**弱**推断信号：评论作者名字模式、少数母语评论、内容/字幕语言。
- **教训**：非拉丁文字脚本检测在养号场景**基本无用**——养号评论清一色写英文。地域更适合让 **Agent 读评论样本做定性判断**（标注为推断），不要硬编码名字→国家的猜测器。

### 2.1 目标 = 辐射欧美：非欧美必须显著标注（硬要求）
- **业务目标是找能辐射欧美的 KOL。** 每次评估**必须明确判定**受众归属，并写入飞书字段 **`受众欧美辐射(推断)`**：`欧美为主 / 分散无主导 / 非欧美为主 / 未知`。
- **判定为"非欧美为主"或"分散无主导"时，必须在报告结论(BLUF)显著标注为风险旗**，不得只埋在受众段；并**显著拉低"受众匹配"维度分**。
- 因是推断，一律标"疑似 + 待 KOL后台真实受众数据 核实"；真值靠邮件环节要 KOL后台真实受众数据。
- 信号：评论敬语("Sir"常见于印度)、用户名地域特征、母语评论、内容语言/字幕语言。
- 现状提醒：已抽样的号普遍偏"非欧美/分散/养号"，**真正欧美为主的号是稀缺的，是找人(discovery)阶段要重点解决的**。

### 2.2 报告「受众画像」区块：能算的算，算不了的诚实标注（对标 Modash 但不造假）
- Modash 的受众画像（性别/年龄/国家%/语言）是**建模的关注者 panel 数据**，公开抓取**做不到**——绝不硬编一个假的出来。
- 我们能真做的：**评论语言分布(估算)**（`youtube_data.py` 输出，对应 Modash 的 commenter-language）+ **受众地域(推断/欧美辐射)**。
- 必须明确标注**做不到**的：性别 / 年龄 / 精确国家% / notable / lookalikes → "需 KOL后台真实受众数据 / Modash"。
- 报告区块用表格分三档：✅评论检测(语言分布) / ⚠️推断(地域/欧美辐射) / ❌需KOL后台真实受众数据·Modash(性别·年龄·国家%·notable)。

### 2.3 创作者国家：curl /about 抠真值（把猜测变核验）
- **yt-dlp 拿不到国家（country 永远空）**；但 **curl 频道 `/about` 页**（浏览器 UA + `Accept-Language: en-US`）→ 正则 `"country":"..."` 能抠到**创作者自报国家**，并能拿 `canonicalBaseUrl` 的标准 @handle。**curl /about 是普通 HTML 请求，不像 yt-dlp 那样被反爬墙限流。** 已实测：Erik=Canada（与 Modash 一致）、ondee=Brazil。`youtube_data.py` 内已实现这段逻辑，写飞书字段 `创作者国家(自报)`。
- **三条必须分清的"地域"**（别再混）：
  1. **创作者国家**（/about，**自报真值**）——可靠但是"创作者声称在哪"。
  2. **受众地域**（评论推断，**我们真正要的"辐射欧美"**）——靠评论语言/名字/敬语。
  3. 受众画像真国家%（KOL后台真实受众数据/Modash）。
- **两个坑**：① **创作者国家 ≠ 受众地域**——实测 RavikanthReddy 创作者自报**美国**，但评论区"Sir"+印度名，受众疑似印度。② **可被养号伪装**——8 个样本里 6 个报"United States"，含明显养号号（AI Future/rimontec/AiFinder），**养号倾向报 US 装正规**。所以创作者国家是**有用锚点，但不能单看**，仍要结合受众推断 + 真实性判断。
- 纯文本判印度会漏一大半（英文简介的印度博主名字简介都看不出）→ 创作者国家用 /about，受众用评论，两条都要。

## 3. 数据获取：yt-dlp 默认 + TikHub 限流兜底（省钱策略）

- 频道指标、视频统计、字幕、**评论** 都能用 yt-dlp 免费抓。
- 商务邮箱常能从频道简介里正则提取（环节②"找邮箱"可部分白嫖）。
- 视频总数：必须**不设 `--playlist-end` 上限**枚举 `/videos`（截断会得 None）。

### 3.1 限流现实（重要）
- **元数据（统计/视频列表/简介）限流风险低**；**评论是重灾区**——抓评论要翻很多页，密集抓会触发 YouTube 按 IP 的反爬墙（"Sign in to confirm you're not a bot"），**一旦触发会连累元数据**，需等数十分钟恢复或带 cookies。
- 缓解：评论抓取已加**节流(视频间 2s)+退避重试(4s/8s)**；大批量评论（受众估算/找人）务必慢抓。
- 应急：`yt-dlp --cookies-from-browser chrome` 用登录态可绕过反爬墙。

### 3.2 抓取策略：yt-dlp 默认 → TikHub 兜底（内置在各平台脚本）
- **默认全走 yt-dlp（免费）**。YouTube 评论抓取在 `data_scrawl/youtube_data.py` 内部先走 yt-dlp，**仅当被限流且配了 `TIKHUB_API_KEY` 时才回退 TikHub**（按积分付费）→ TikHub 只在真限流时花钱，成本最小。
- 没配 key 时回退自动跳过，如实返回失败（不乱花钱）。
- TikHub 真正增量价值：**按关键词搜频道（找人）** + 大规模抓评论 + 限流兜底。它解决"抓得稳/不限流"，**不解决受众画像真值**（那是 Modash）。
- ✅ TikHub 评论字段映射**已用真 key 核对通过**：`data.comments[].{content(文本), author.display_name(作者), like_count(字符串)}`；接口返回 `message:"...will incur a charge"` 确认按次计费。闭环已实测：yt-dlp 被墙→自动回退 TikHub→正常出信号。
- 省钱优化：回退 TikHub 时**跳过"最新评论"二次采样**（同批数据，省一次积分）；最新采样只在免费 yt-dlp 上做。
- **TikHub 限 10 RPS**：各平台脚本内置进程内限速（最小间隔 0.12s ≈ 8 RPS，留余量），实测达标。⚠️注意：限速是**进程内**的——若将来"找人/批量"用**多进程并发**跑 TikHub，各进程不共享计数，可能合计超 10 RPS；那时需改为**单进程内并发 + 共享限速器**，或用信号量协调。
- **语言 ≠ 地域**：评论语言分布只反映"用什么语言写"，不等于受众在哪。如 RavikanthReddy 评论 100% 英文，但"Sir"敬语+用户名显示受众疑似印度——**印度受众用英文评论**。语言分布能识破非英语受众(巴西/西语等)，但识别不了"英文书写的非欧美受众"，后者仍靠名字/敬语推断。

## 3.5 评论购买意图：必须多语言（交叉验证教训）
- 评论购买意图（"vou testar""usei…muito bom"）是**转化潜力的强信号**，但**纯英文正则会对非英语频道完全失灵**。
- 实测教训：对巴西号 @ondeeuclico，英文正则报"意图 0%"，实际评论全是真实葡语购买/试用意图 → 严重误判。
- 现状：`youtube_data.py` 内的购买意图规则已覆盖 **英/葡/西**，并输出"评论语言提示"。其他语言（日/阿/印地…）规则仍失灵，**需模型分类**。
- 通用原则：**任何基于关键词的英文规则，遇到非英语频道都要先看"评论语言提示"再决定是否采信**；不确定就让 Agent 读样本判断。

## 4. 评估产出形态
- **结论先行（BLUF）**：报告**最顶部**放"结论摘要"——判定(合作/观望/放弃) + 5 维评分表 + 一句话理由 + 硬证据/关键旗子 + 报价一行；想深究再看下方明细。让人 3 秒看懂"做不做、为什么"。
- 永远输出**结构化判断卡**：各维度分 + 理由 + 异常标记 + 对照基准 + 结论(合作/观望/放弃)。
- 绝不只甩一个分数或"健康/不健康"。
- 评估粒度 = **KOL × 业务线**（同一 KOL 对不同业务线结论可能不同；一个号可对 Bloome 和 Renoise 各打一次分、各出一张卡）。
- **5 维评分（canonical，每维 0-10）**：受众匹配 / 内容承载 / 流量稳定 / 互动可信 / 报价合理。
  - 来源标注：受众匹配·流量稳定·互动可信·报价合理 = 脚本自动可得；**内容承载（能否讲清/demo 产品）需 Gemini 看视频**；受众地域真值需 KOL后台真实受众数据。
- **业务匹配标准**（Bloome / Renoise 的受众/关键词/加减分/CPM/标杆号）见飞书「业务线评估标准表」，是判断卡受众匹配 + 内容承载维度的对照基准。

### 4.1 铁律：业务匹配 ≠ 该合作（闸门一票否决题材匹配）
- 题材关键词只决定"**方向对不对**"（适合 Bloome 还是 Renoise）；**真实性/受众闸门才决定"能不能合作"**。
- 实证：Future AI(@AIFutureofficial01) 题材完全像 Bloome（AI 工具/agent），关键词匹配会判它为 Bloome 向——但养号 + 非欧美 → **放弃**。**先过闸门（真实+在市场），再谈题材匹配与定价。**

## 5. 实战案例：@AIFutureofficial01（红灯，建议放弃）
- 表象：44.3K 订阅、加权 ER 6.18%、AI 工具盘点、全英文、每 2 天更。
- 真相（评论分析 240 条）：泛泛灌水 79%、**模板化重复评论 6 组**、作者跨视频重复（6×/4×）、频道主 0 回复 → **互动严重灌水 / 疑似养号**。
- 受众：名字以南亚/非洲为主 + 夹杂西语，**无欧美核心受众迹象** → 与拉新/转化欧美目标不符。
- 结论：**光看订阅数和 ER 会被骗，是评论分析暴露了真相。** 这类号应在初筛即筛掉。

### 5.1 ⚠️ 真实合作教训：Shark Numbers（@IvanKv）—— 被"伪深度评论"骗过
- **真实结果**：实际合作后获客仅 **200 多个**（1.84M 订阅 / 中位 70 万播放的号，转化惨败）。
- **我的初版评估也被骗了**：评论"读起来有深度"（讨论 agent/AI build），我一度给"互动可信"打 8 分。
- **复盘发现的两条铁律（已固化为信号）**：
  1. **播放方差**：合作期视频播放全在 30 万上下、整齐划一 → 注水（注水常只在投放窗口出现，须按合作窗口查；CV 偏低 + 几乎无低播放也可疑）。
  2. **评论语言学**：实测 180 条评论 **疑问句 4% / 表情 3% / 时间戳 3% / 纯陈述句 91%**。真观众会提问、发表情、带时间戳；**全是打磨过的陈述句 = 脚本化/AI 评论**。
- **教训**：评论"有没有内容"不够，要看"**像不像真观众的行为**"。陈述句能伪装深度，疑问/表情/时间戳难伪装。这是 Modash 类平台也给不了的判断。

## 6. 报价评估（定价准则）

**原则：不按订阅数报价。** 按 `有效播放 × 受众匹配 × 商业风险`，且这三者要**算进数字**，不能停在定性。

### 6.1 计算链
1. **有效播放** = 真实中位播放 × **真实度系数**
   - 中位播放：近 8-12 条**长视频**（排除 Shorts，除非合作形式就是 Shorts），用中位数不用均值；近期下滑则按 30-60 天加权；老爆款只算潜力不算依据。
   - 真实度系数（判断值，未校准）：真实 ×1.0 ／ 存疑 ×0.4-0.6 ／ 养号 ×0.1-0.2。**养号的名义播放按 CPM 看着正常，必须先打折再算。**
2. **可接受 CPM** = 品类 CPM × **地域系数**
   - 品类：泛 AI 工具 $40-80 ／ 开发者·AI Agent·SaaS·B2B 高意图 $80-150（针对英语/欧美受众）。>$200 需转化案例/社区/newsletter 背书，否则砍价或 pass。
   - 地域系数：欧美/英语 ×1.0 ／ 拉美·葡西 ×0.35（且需本地化落地页+coupon）／ 印度·东南亚 ×0.3（付费能力与转化路径单独看）。
3. **合理报价区间** = 有效播放 × 可接受 CPM / 1000
   - **极低播放（<1-2K）CPM 公式失真** → 按 flat fee / 产品置换处理。
4. **CAC 上限（占位）**：以上是**供给侧公允价**，回答"市场价公不公道"，≠"对我们划算"。真正天花板 = `目标CAC × CTR × 注册率 × 付费率`，需首测跑出真实 CAC 后才能算，届时覆盖市场价。

### 6.2 合作形式与议价信号
- 形式价值：60s 插播 < 独立评测/教程（能演示真实 workflow 更高）< 描述区前两行/置顶评论/社区帖/newsletter/Discord 分发。二次投流/素材/排他**另算钱**。
- 议价空间信号（已抓数据可反推）：只自推无品牌单的号→无既定报价、好谈；简介挂商务邮箱→公开求单；播放/订阅比衰减→若按订阅开价即虚高；更新极高频→库存多更可谈。

### 6.3 三档决策
直接做（CPM 合理+受众精准+互动真实）／ 砍价做（方向合适但播放不稳或报价偏高）／ 不建议（靠老爆款·订阅撑价、近期中位播放低、或互动不可信）。

### 6.4 必须沉淀：从第一单起攒自有定价数据
框架现在全锚在行业 CPM 带上 = 永远在"猜"。**每次议价都记 `开价/还价/成交价 + 计价口径 + 当时指标`**，攒几十条拟合出我们自己的定价曲线，比行业带准。字段见 `feishu-schema.md` 评估记录表。

## 7. 信号≠判决：靠真实结果验证（合作标注集）

- **没有一条信号能单独定性**。判断靠**多信号叠加**（硬信号定罪、软信号当提示灯）+ **相对同类对照**，最终由人确认。
- **软信号（语言学等）只指向"该看哪里"，不自动评分**；硬信号（重复评论、跨视频刷账号）才可作较强依据。
